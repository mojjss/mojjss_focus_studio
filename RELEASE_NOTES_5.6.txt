Mojjss Focus Studio v5.6 — Stability Update
Released: 2026-07-18

NEW
- Diagnostics page inside the desktop application.
- Compact tunnel status in the sidebar.
- Periodic checks for the Cloudflared Windows service, local camera origin,
  public camera route, dashboard health, and optional SYSTEM watchdog.
- Non-modal disconnect and recovery notifications.
- Optional automatic tunnel watchdog, installed as a SYSTEM Scheduled Task.
- One-click launch buttons for diagnosis, tunnel repair, and watchdog setup.

FIXED
- Normal browser/client disconnects no longer create duplicate WinError 10053
  tracebacks in the local monitor server.
- Diagnostic and repair BAT files now resolve project paths safely, including
  paths containing spaces and trailing separators.
- The full diagnosis separates active Cloudflared crashes from historical
  service events instead of treating every old event as a current failure.
- Tunnel diagnosis distinguishes error 1033 from origin-side 502 errors.
- The repair tool can store a current tunnel token in a restricted token file
  on supported cloudflared versions instead of leaving it in the service
  command line.

KEPT FROM v5.5.2
- 1-day and 3-day analytics/session ranges.
- Session pagination patch.
- Removed temporary D1 debug logging.
- Data-preserving updater.

PRIVACY AND NETWORK SAFETY
- No config.json, database, schedule, timer state, token, key, password, export,
  or camera image is included in this release archive.
- Diagnosis reports are automatically redacted.
- The watchdog and diagnosis tools do not reset v2rayN, Tailscale, Winsock,
  adapters, DNS, firewall rules, or Windows proxy settings.
- Never share the long Cloudflare tunnel token beginning with eyJ.
